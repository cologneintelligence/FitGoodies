This package resolves cross references. In some circumstances you cannot predict a value, but it is returned in an earlier test case. Or sometimes you just want to know whether a value is not null. fitgoodies.references allow you to register a "cross reference processor" which is called before the final comparison. It enables the user to manipulate the cell's content and makes the comparison a bit more flexible.

Some processors are already available: the {@link fitgoodies.references.processors.EmptyCrossReferenceProcessor} checks whether a value is null or "not null", the {@link fitgoodies.references.processors.StorageCrossReferenceProcessor} gives you the opportunity to save and restore values in variables, the {@link fitgoodies.references.processors.PropertyCrossReferenceProcessor} makes it possible to read property files and the {@link fitgoodies.references.processors.FileFixtureCrossReferenceProcessor} allows to search for files (see {@link fitgoodies.file.FileFixture} for more information).

Own processors can be implemented by extending the {@link fitgoodies.references.processors.AbstractCrossReferenceProcessor}. They must be registered in the {@link fitgoodies.references.CrossReferenceHelper}, either directly, or via HTML using the {@link fitgoodies.references.SetupFixture}.
