TypeAdapters are used by fit to convert a String (which comes from your tables) into an Object (using {@link fitgoodies.adapters.AbstractTypeAdapter}'s parse method), convert an Object into its representing String (using {@link fitgoodies.adapters.AbstractTypeAdapter#toString(Object)}) and checking whether two Objects are equal (using {@link fitgoodies.adapters.AbstractTypeAdapter}'s equal method).

TypeAdapters for classes which fit can't parse can be registered from Source using {@link fitgoodies.adapters.TypeAdapterHelper#register(Class)} or by your HTML input using the {@link fitgoodies.adapters.SetupFixture}.
The TypeAdapters which come with fitgoodies are automatically registered.

If everything except parsing works fine, you do not need to implement a TypeAdapter. You can then register a parser. See {@link fitgoodies.parsers} for more information.
